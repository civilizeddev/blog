TypeScript 시작하기

April 16, 2020

Hyeonsoo Lee

반여동 뚜껑머리

안녕하세요, 반여동 뚜껑머리입니다.

오늘은 TypeScript를 준비했으니 한 번 구경하고 가세요.

#준비물

  • 맥북
note

Windows로 설명하려니 귀찮습니다. 그냥 macOS 기준으로 설명하겠습니다.

#프로젝트 초기화

note

쭉 따라서 실행해보십시오.

# Homebrew 설치
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"
# Node Version Manager 설치
$ brew install nvm
# Node.js 12.x (LTS) 설치
$ nvm install 12
# Yarn (Package Manager) 설치
$ brew install yarn --ignore-dependencies
# Node 프로젝트 초기화
$ yarn init
# TypeScript 설치
$ yarn add typescript --dev
# TypeScript 설정 초기화
$ tsc --init

#tsconfig.json 설정

tsc --init을 실행하면 tsconfig.json이 생성됩니다.

TypeScript 프로젝트를 어떻게 빌드할지 설정하는 파일입니다.

각 옵션 필드의 내용은 아래를 참조하십시오.

tsconfig 설정 목록

저는 tsconfig.json과 tsconfig.prod.json을 선언해서 사용합니다.

tsconfig.json
{
"compilerOptions": {
"target": "ES2019",
"module": "commonjs",
"outDir": "./lib",
"rootDir": "./src",
"strict": true,
"moduleResolution": "node",
"esModuleInterop": true,
"experimentalDecorators": true,
"emitDecoratorMetadata": true,
"resolveJsonModule": true,
"skipLibCheck": true,
"forceConsistentCasingInFileNames": true
}
}
tsconfig.prod.json
{
"extends": "./tsconfig.json",
"include": [
"src/**/*.ts"
],
"exclude": [
"src/**/*.test.ts"
]
}

각각 어떤 설정인지 자세히 알아보세요.

tip

compilerOptions.target은 Node.js 런타임의 버전에 따라 지정합니다.

Node.js 12.x 버전은 현재 ECMA2019까지 구현하고 있습니다.

#빌드

note

TypeScript의 빌드 과정은 ts 파일을 js 파일과 d.ts 파일로 변환합니다.

tsconfig.json는 기본값
$ tsc --build --force .
tsconfig.prod.json를 사용하려면 파일 이름 명시
$ tsc --build --force tsconfig.prod.json
tip

package.json 파일의 scripts 항목에 선언해 놓으면 다음에 사용하기 편합니다.

{
"scripts": {
"build": "tsc --build --force .",
"build:prod": "tsc --build --force tsconfig.prod.json",
...
},
...
}

#실행

note

tsconfig.json에서 compilerOptions.outDir에 ./lib 경로를 지정했기 때문에 해당 경로에 js, d.ts 파일이 생성됩니다.

경로를 지정하면 해당 디렉터리의 index.js를 먼저 실행합니다.

$ node lib/

index.js 대신 다른 파일을 실행하려면 파일 이름을 명시적으로 지정합니다.

$ node lib/server.js
tip

package.json 파일의 main 항목을 지정하면 다음에 사용하기 편합니다.

{
"main": "lib/index.js"
}

그러면 이렇게 실행할 수 있습니다.

$ node .

#빌드 & 실행 한 방에

일일이 빌드, 실행 과정을 수동으로 반복하는 것은 괴롭습니다.

아예 ts 파일을 js 파일처럼 읽어서 바로 실행해주는 ts-node 모듈이 있습니다.

$ yarn add ts-node --dev

이제 그냥 이렇게 하면 됩니다.

$ ts-node src/index.ts

#Watch

파일의 업데이트를 감지하여 자동으로 빌드를 재실행하는 기능을 watch라고 합니다.

ts-node-dev라는 모듈을 사용해보겠습니다.

note

ts-node-dev = ts-node + node-dev

$ yarn add ts-node-dev --dev
$ ts-node-dev --respawn ./src
tip

package.json 파일의 scripts 항목에 선언해 놓으면 다음에 사용하기 편합니다.

{
"scripts": {
"debug": "ts-node-dev --respawn ./src",
...
},
...
}

#잘 되나요?

src/index.ts
console.log('Hello, World!')
$ yarn debug
Hello, World!
